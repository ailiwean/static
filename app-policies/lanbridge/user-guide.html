Network Share User Guide

Use Network Share as a LAN proxy gateway; clients must manually set host IP + port (no auto-discovery/encryption).

Purpose

Use this device as a LAN proxy gateway on the same Wi‑Fi/hotspot; other devices forward traffic via this host (can stack with host VPN). Auto-discovery/encryption removed; clients must fill host IP and port manually.

Host steps

  1. Open the app, pick proxy mode (recommend “Auto-detect”) and port (default 1080). Current version forces full traffic via this proxy.
  2. Ensure host and clients are on the same LAN; port not occupied.
  3. Tap “Start Gateway” and keep the foreground service alive.

Client setup (manual IP + port)

  • Android: Wi‑Fi details → Proxy = Manual; host = this device IP, port = app port. Or set HTTP/HTTPS in apps; Auto-detect mode typically works with HTTP/HTTPS/SOCKS5.
  • iOS: Settings → Wi‑Fi → current network → Configure Proxy = Manual; server = host IP, port = app port (HTTP/HTTPS).
  • Windows: Settings → Network & Internet → Proxy → Manual; address = host IP, port = app port (HTTP/HTTPS); apps that support SOCKS5 can set SOCKS5.
  • macOS: System Settings → Network → current interface → Details → Proxies; check HTTP/HTTPS (or SOCKS5), server = host IP, port = app port.
  • Other devices: fill host IP + port in HTTP/HTTPS/SOCKS5 proxy settings supported by the device/app.

作用

同一 Wi‑Fi/热点下，将本机作为代理网关，其他设备通过本机转发流量（可叠加本机 VPN）。已移除自动发现/加密，需手动填写主机 IP 和端口。

主机端步骤

  1. 打开 App，选代理模式（推荐“自动识别”）和端口（默认 1080），当前版本全量经本机代理。
  2. 确认主机与客户端同局域网，端口未被占用。
  3. 点击“启动网关”，保持前台服务运行。

客户端配置（手动 IP + 端口）

  • Android：Wi‑Fi 详情→代理=手动，主机名填本机 IP，端口填 App 端口；或在支持代理的应用中填写 HTTP/HTTPS（自动识别通常兼容 HTTP/HTTPS/SOCKS5）。
  • iOS：设置→Wi‑Fi→当前网络→配置代理=手动，服务器填本机 IP，端口填 App 端口（HTTP/HTTPS）。
  • Windows：设置→网络和 Internet→代理→手动代理，地址=本机 IP，端口=App 端口（HTTP/HTTPS）；支持 SOCKS5 的应用可直接填 SOCKS5。
  • macOS：系统设置→网络→当前接口→详情→代理，勾选 HTTP/HTTPS（或 SOCKS5），服务器=本机 IP，端口=App 端口。
  • 其他设备：在支持的 HTTP/HTTPS/SOCKS5 代理入口手动填本机 IP + 端口。

الغرض

استخدم الجهاز كبوابة وكيل داخل نفس شبكة Wi‑Fi/نقطة الاتصال؛ تُمرَّر حركة الأجهزة الأخرى عبر هذا المضيف (يمكن تكديس VPN المضيف). تمت إزالة الاكتشاف التلقائي/التشفير؛ يجب على العميل إدخال IP المضيف والمنفذ يدويًا.

خطوات المضيف

  1. افتح التطبيق، واختر وضع الوكيل (يُنصح بـ “اكتشاف تلقائي”) والمنفذ (الافتراضي 1080). الإصدار الحالي يوجّه كل الحركة عبر الوكيل.
  2. تأكد أن المضيف والعميل على نفس شبكة LAN وأن المنفذ غير مشغول.
  3. اضغط “بدء البوابة” وابقَ على الخدمة في الواجهة.

إعداد العميل (IP + منفذ يدوي)

  • Android: تفاصيل Wi‑Fi → الوكيل = يدوي؛ الخادم = IP لهذا الجهاز، المنفذ = منفذ التطبيق. أو املأ HTTP/HTTPS في التطبيقات؛ وضع الاكتشاف التلقائي يدعم عادة HTTP/HTTPS/SOCKS5.
  • iOS: الإعدادات → Wi‑Fi → الشبكة الحالية → تكوين الوكيل = يدوي؛ الخادم = IP المضيف، المنفذ = منفذ التطبيق (HTTP/HTTPS).
  • Windows: الإعدادات → الشبكة والإنترنت → الوكيل → يدوي؛ العنوان = IP المضيف، المنفذ = منفذ التطبيق (HTTP/HTTPS)؛ التطبيقات التي تدعم SOCKS5 يمكنها ضبط SOCKS5.
  • macOS: إعدادات النظام → الشبكة → الواجهة الحالية → التفاصيل → الوكلاء؛ فعّل HTTP/HTTPS (أو SOCKS5)، الخادم = IP المضيف، المنفذ = منفذ التطبيق.
  • أجهزة أخرى: أدخل IP المضيف + المنفذ في إعدادات وكيل HTTP/HTTPS/SOCKS5 المدعومة.

Zweck

Nutze dieses Gerät im selben Wi‑Fi/Hotspot als LAN‑Proxy-Gateway; andere Geräte leiten den Verkehr über diesen Host (VPN des Hosts kann gestapelt werden). Auto-Discovery/Verschlüsselung wurden entfernt; Clients müssen Host-IP und Port manuell eintragen.

Schritte auf dem Host

  1. App öffnen, Proxy-Modus wählen (empfohlen „Automatisch erkennen“) und Port wählen (Standard 1080). Aktuelle Version routet den gesamten Verkehr über den Proxy.
  2. Sicherstellen, dass Host und Clients im selben LAN sind und der Port frei ist.
  3. „Gateway starten“ tippen und den Vordergrunddienst aktiv lassen.

Client-Konfiguration (manuelle IP + Port)

  • Android: WLAN-Details → Proxy = Manuell; Host = IP dieses Geräts, Port = App-Port. Oder HTTP/HTTPS in Apps eintragen; Auto-Erkennung funktioniert meist mit HTTP/HTTPS/SOCKS5.
  • iOS: Einstellungen → WLAN → aktuelles Netzwerk → Proxy konfigurieren = Manuell; Server = Host-IP, Port = App-Port (HTTP/HTTPS).
  • Windows: Einstellungen → Netzwerk & Internet → Proxy → Manuell; Adresse = Host-IP, Port = App-Port (HTTP/HTTPS); Apps mit SOCKS5-Unterstützung können SOCKS5 setzen.
  • macOS: Systemeinstellungen → Netzwerk → aktuelle Schnittstelle → Details → Proxys; HTTP/HTTPS (oder SOCKS5) markieren, Server = Host-IP, Port = App-Port.
  • Andere Geräte: Host-IP + Port in den HTTP/HTTPS/SOCKS5-Proxyfeldern des Geräts/der App eintragen.

Propósito

Usa este dispositivo como puerta de enlace proxy en la misma red Wi‑Fi/punto de acceso; los demás equipos envían su tráfico a través de este host (puede combinarse con la VPN del host). Se eliminó el autodescubrimiento/cifrado; el cliente debe ingresar manualmente la IP y el puerto del host.

Pasos en el host

  1. Abre la app, elige el modo de proxy (se recomienda “Detección automática”) y el puerto (predeterminado 1080). La versión actual dirige todo el tráfico por el proxy.
  2. Asegúrate de que el host y los clientes estén en la misma LAN y que el puerto esté libre.
  3. Toca “Iniciar gateway” y mantén vivo el servicio en primer plano.

Configuración del cliente (IP + puerto manual)

  • Android: Detalles de Wi‑Fi → Proxy = Manual; host = IP de este dispositivo, puerto = puerto de la app. O configura HTTP/HTTPS en las apps; la detección automática suele funcionar con HTTP/HTTPS/SOCKS5.
  • iOS: Ajustes → Wi‑Fi → red actual → Configurar proxy = Manual; servidor = IP del host, puerto = puerto de la app (HTTP/HTTPS).
  • Windows: Configuración → Red e Internet → Proxy → Manual; dirección = IP del host, puerto = puerto de la app (HTTP/HTTPS); las apps que soporten SOCKS5 pueden usar SOCKS5.
  • macOS: Ajustes del sistema → Red → interfaz actual → Detalles → Proxies; marca HTTP/HTTPS (o SOCKS5), servidor = IP del host, puerto = puerto de la app.
  • Otros dispositivos: introduce IP del host + puerto en los ajustes de proxy HTTP/HTTPS/SOCKS5 que el equipo o la app admitan.

هدف

از این دستگاه در همان شبکه Wi‑Fi/هات‌اسپات به‌عنوان درگاه پروکسی LAN استفاده کنید؛ ترافیک سایر دستگاه‌ها از این میزبان عبور می‌کند (می‌توان آن را با VPN میزبان ترکیب کرد). کشف خودکار/رمزگذاری حذف شده است؛ کلاینت باید IP میزبان و پورت را دستی وارد کند.

مراحل میزبان

  1. اپ را باز کنید، حالت پروکسی (پیشنهاد «تشخیص خودکار») و پورت (پیش‌فرض 1080) را برگزینید. نسخه فعلی تمام ترافیک را از طریق پروکسی هدایت می‌کند.
  2. مطمئن شوید میزبان و کلاینت در یک LAN هستند و پورت اشغال نشده است.
  3. روی «شروع درگاه» بزنید و سرویس پیش‌زمینه را فعال نگه دارید.

پیکربندی کلاینت (IP + پورت دستی)

  • Android: جزئیات Wi‑Fi → پروکسی = دستی؛ سرور = IP این دستگاه، پورت = پورت اپ. یا در اپ‌های پشتیبان، HTTP/HTTPS را پر کنید؛ حالت تشخیص خودکار معمولاً با HTTP/HTTPS/SOCKS5 کار می‌کند.
  • iOS: تنظیمات → Wi‑Fi → شبکه فعلی → پیکربندی پروکسی = دستی؛ سرور = IP میزبان، پورت = پورت اپ (HTTP/HTTPS).
  • Windows: تنظیمات → شبکه و اینترنت → پروکسی → دستی؛ آدرس = IP میزبان، پورت = پورت اپ (HTTP/HTTPS)؛ اپ‌های دارای SOCKS5 می‌توانند SOCKS5 را تنظیم کنند.
  • macOS: تنظیمات سیستم → شبکه → رابط فعلی → جزئیات → پروکسی‌ها؛ HTTP/HTTPS (یا SOCKS5) را تیک بزنید، سرور = IP میزبان، پورت = پورت اپ.
  • دستگاه‌های دیگر: در تنظیمات پروکسی HTTP/HTTPS/SOCKS5 پشتیبانی‌شده، IP میزبان + پورت را وارد کنید.

Objet

Utilisez cet appareil comme passerelle proxy LAN sur le même Wi‑Fi/point d’accès ; les autres appareils font transiter leur trafic via cet hôte (VPN du serveur hôte possible). La découverte automatique/chiffrement a été retirée ; le client doit saisir manuellement l’IP et le port de l’hôte.

Étapes côté hôte

  1. Ouvrez l’app, choisissez le mode proxy (recommandé « Détection auto ») et le port (1080 par défaut). La version actuelle force tout le trafic via ce proxy.
  2. Vérifiez que l’hôte et les clients sont sur le même LAN et que le port est libre.
  3. Appuyez sur « Démarrer la passerelle » et maintenez le service au premier plan.

Configuration client (IP + port manuels)

  • Android : Détails Wi‑Fi → Proxy = Manuel ; hôte = IP de cet appareil, port = port de l’app. Ou renseignez HTTP/HTTPS dans les apps ; le mode Détection auto fonctionne généralement avec HTTP/HTTPS/SOCKS5.
  • iOS : Réglages → Wi‑Fi → réseau actuel → Configurer le proxy = Manuel ; serveur = IP hôte, port = port de l’app (HTTP/HTTPS).
  • Windows : Paramètres → Réseau et Internet → Proxy → Manuel ; adresse = IP hôte, port = port de l’app (HTTP/HTTPS) ; les apps compatibles peuvent régler SOCKS5.
  • macOS : Réglages Système → Réseau → interface actuelle → Détails → Proxys ; cochez HTTP/HTTPS (ou SOCKS5), serveur = IP hôte, port = port de l’app.
  • Autres appareils : renseignez IP hôte + port dans les paramètres proxy HTTP/HTTPS/SOCKS5 pris en charge.

उद्देश्य

उसी Wi‑Fi/हॉटस्पॉट में इस डिवाइस को LAN प्रॉक्सी गेटवे बनाएं; अन्य डिवाइस का ट्रैफ़िक इस होस्ट से होकर गुजरता है (होस्ट VPN को जोड़ सकते हैं)। ऑटो-डिस्कवरी/एन्क्रिप्शन हटा दिया गया है; क्लाइंट को होस्ट IP और पोर्ट मैन्युअल भरना होगा।

होस्ट के चरण

  1. ऐप खोलें, प्रॉक्सी मोड चुनें (अनुशंसित “ऑटो-डिटेक्ट”) और पोर्ट चुनें (डिफ़ॉल्ट 1080)। वर्तमान संस्करण पूरा ट्रैफ़िक प्रॉक्सी से भेजता है।
  2. सुनिश्चित करें कि होस्ट और क्लाइंट एक ही LAN पर हैं और पोर्ट खाली है।
  3. “गेटवे शुरू करें” टैप करें और फोरग्राउंड सेवा को चालू रखें।

क्लाइंट सेटअप (मैन्युअल IP + पोर्ट)

  • Android: Wi‑Fi विवरण → Proxy = Manual; होस्ट = इस डिवाइस का IP, पोर्ट = ऐप पोर्ट। या ऐप में HTTP/HTTPS भरें; ऑटो-डिटेक्ट मोड आम तौर पर HTTP/HTTPS/SOCKS5 के साथ काम करता है।
  • iOS: Settings → Wi‑Fi → वर्तमान नेटवर्क → Configure Proxy = Manual; सर्वर = होस्ट IP, पोर्ट = ऐप पोर्ट (HTTP/HTTPS)।
  • Windows: Settings → Network & Internet → Proxy → Manual; पता = होस्ट IP, पोर्ट = ऐप पोर्ट (HTTP/HTTPS); SOCKS5 सपोर्ट करने वाले ऐप्स में सीधे SOCKS5 सेट करें।
  • macOS: System Settings → Network → वर्तमान इंटरफ़ेस → Details → Proxies; HTTP/HTTPS (या SOCKS5) चुनें, सर्वर = होस्ट IP, पोर्ट = ऐप पोर्ट।
  • अन्य डिवाइस: समर्थित HTTP/HTTPS/SOCKS5 प्रॉक्सी सेटिंग में होस्ट IP + पोर्ट भरें।

Tujuan

Gunakan perangkat ini sebagai gateway proxy LAN pada Wi‑Fi/hotspot yang sama; perangkat lain meneruskan trafik lewat host ini (bisa ditumpuk dengan VPN host). Fitur auto-discovery/enkripsi dihapus; klien harus mengisi IP host dan port secara manual.

Langkah di host

  1. Buka app, pilih mode proxy (disarankan “Deteksi otomatis”) dan port (default 1080). Versi sekarang memaksa seluruh trafik lewat proxy.
  2. Pastikan host dan klien berada pada LAN yang sama dan port tidak dipakai.
  3. Ketuk “Mulai Gateway” dan biarkan layanan foreground tetap aktif.

Pengaturan klien (IP + port manual)

  • Android: Detail Wi‑Fi → Proxy = Manual; host = IP perangkat ini, port = port app. Atau isi HTTP/HTTPS di aplikasi; mode deteksi otomatis umumnya bekerja dengan HTTP/HTTPS/SOCKS5.
  • iOS: Settings → Wi‑Fi → jaringan saat ini → Configure Proxy = Manual; server = IP host, port = port app (HTTP/HTTPS).
  • Windows: Settings → Network & Internet → Proxy → Manual; address = IP host, port = port app (HTTP/HTTPS); aplikasi yang mendukung SOCKS5 bisa mengatur SOCKS5.
  • macOS: System Settings → Network → antarmuka saat ini → Details → Proxies; centang HTTP/HTTPS (atau SOCKS5), server = IP host, port = port app.
  • Perangkat lain: isi IP host + port di pengaturan proxy HTTP/HTTPS/SOCKS5 yang didukung.

目的

同一の Wi‑Fi/ホットスポット内で本機を LAN プロキシ・ゲートウェイとして使用し、他のデバイスの通信を本機経由で転送します（ホストの VPN と併用可）。自動検出/暗号化は削除されているため、クライアントはホスト IP とポートを手動入力する必要があります。

ホストの手順

  1. アプリを開き、プロキシモード（推奨「自動識別」）とポート（デフォルト 1080）を選択します。現行バージョンはすべてのトラフィックをプロキシ経由にします。
  2. ホストとクライアントが同じ LAN 上にあり、ポートが空いていることを確認します。
  3. 「ゲートウェイ開始」をタップし、フォアグラウンドサービスを維持します。

クライアント設定（手動 IP + ポート）

  • Android: Wi‑Fi 詳細 → プロキシ = 手動；ホスト = 本機の IP、ポート = アプリのポート。あるいはアプリの HTTP/HTTPS に設定。自動識別モードは通常 HTTP/HTTPS/SOCKS5 に対応します。
  • iOS: 設定 → Wi‑Fi → 使用中のネットワーク → プロキシを構成 = 手動；サーバ = ホスト IP、ポート = アプリのポート（HTTP/HTTPS）。
  • Windows: 設定 → ネットワークとインターネット → プロキシ → 手動プロキシ；アドレス = ホスト IP、ポート = アプリのポート（HTTP/HTTPS）。SOCKS5 対応アプリは SOCKS5 を設定可。
  • macOS: システム設定 → ネットワーク → 使用中のインターフェース → 詳細 → プロキシ；HTTP/HTTPS（または SOCKS5）をチェック、サーバ = ホスト IP、ポート = アプリのポート。
  • その他のデバイス: デバイス/アプリの HTTP/HTTPS/SOCKS5 プロキシ欄にホスト IP とポートを入力します。

목적

동일한 Wi‑Fi/핫스팟에서 이 기기를 LAN 프록시 게이트웨이로 사용하여 다른 장치의 트래픽을 이 호스트로 전달합니다(호스트 VPN과 함께 사용 가능). 자동 검색/암호화 기능은 제거되었으며, 클라이언트가 호스트 IP와 포트를 수동으로 입력해야 합니다.

호스트 단계

  1. 앱을 열어 프록시 모드(“자동 감지” 권장)와 포트(기본 1080)를 선택합니다. 현재 버전은 전체 트래픽을 프록시로 보냅니다.
  2. 호스트와 클라이언트가 같은 LAN에 있고 포트가 비어 있는지 확인합니다.
  3. “게이트웨이 시작”을 눌러 포그라운드 서비스를 유지합니다.

클라이언트 설정(수동 IP + 포트)

  • Android: Wi‑Fi 세부정보 → 프록시 = 수동; 호스트 = 이 기기 IP, 포트 = 앱 포트. 또는 앱의 HTTP/HTTPS 항목에 입력; 자동 감지 모드는 일반적으로 HTTP/HTTPS/SOCKS5와 호환됩니다.
  • iOS: 설정 → Wi‑Fi → 현재 네트워크 → 프록시 구성 = 수동; 서버 = 호스트 IP, 포트 = 앱 포트(HTTP/HTTPS).
  • Windows: 설정 → 네트워크 및 인터넷 → 프록시 → 수동; 주소 = 호스트 IP, 포트 = 앱 포트(HTTP/HTTPS). SOCKS5를 지원하는 앱은 SOCKS5를 설정합니다.
  • macOS: 시스템 설정 → 네트워크 → 현재 인터페이스 → 세부사항 → 프록시; HTTP/HTTPS(또는 SOCKS5) 선택, 서버 = 호스트 IP, 포트 = 앱 포트.
  • 기타 장치: 기기/앱에서 지원하는 HTTP/HTTPS/SOCKS5 프록시 설정에 호스트 IP + 포트를 입력합니다.

Tujuan

Gunakan peranti ini sebagai gerbang proksi LAN pada Wi‑Fi/hotspot yang sama; trafik peranti lain dihantar melalui hos ini (boleh digabung dengan VPN hos). Auto-carikan/penyulitan telah dibuang; klien mesti isi IP hos dan port secara manual.

Langkah hos

  1. Buka app, pilih mod proksi (disyorkan “Auto-detect”) dan port (lalai 1080). Versi semasa memaksa semua trafik melalui proksi.
  2. Pastikan hos dan klien berada dalam LAN yang sama dan port tidak digunakan.
  3. Ketik “Mulakan Gateway” dan kekalkan perkhidmatan latar depan.

Konfigurasi klien (IP + port manual)

  • Android: Butiran Wi‑Fi → Proksi = Manual; hos = IP peranti ini, port = port app. Atau isikan HTTP/HTTPS dalam app; mod auto-detect biasanya berfungsi dengan HTTP/HTTPS/SOCKS5.
  • iOS: Settings → Wi‑Fi → rangkaian semasa → Configure Proxy = Manual; server = IP hos, port = port app (HTTP/HTTPS).
  • Windows: Settings → Network & Internet → Proxy → Manual; alamat = IP hos, port = port app (HTTP/HTTPS); app yang menyokong SOCKS5 boleh menetapkan SOCKS5.
  • macOS: System Settings → Network → antara muka semasa → Details → Proxies; tandakan HTTP/HTTPS (atau SOCKS5), server = IP hos, port = port app.
  • Peranti lain: isi IP hos + port dalam tetapan proksi HTTP/HTTPS/SOCKS5 yang disokong.

  2. Certifique-se de que host e clientes estejam na mesma LAN e que a porta não esteja ocupada.
  3. Toque em “Iniciar gateway” e mantenha o serviço em primeiro plano ativo.

Configuração do cliente (IP + porta manual)

  • Android: Detalhes do Wi‑Fi → Proxy = Manual; host = IP deste dispositivo, porta = porta do app. Ou preencha HTTP/HTTPS nos apps; o modo de detecção automática geralmente funciona com HTTP/HTTPS/SOCKS5.
  • iOS: Ajustes → Wi‑Fi → rede atual → Configurar Proxy = Manual; servidor = IP do host, porta = porta do app (HTTP/HTTPS).
  • Windows: Configurações → Rede e Internet → Proxy → Manual; endereço = IP do host, porta = porta do app (HTTP/HTTPS); apps que suportam SOCKS5 podem definir SOCKS5.
  • macOS: Ajustes do Sistema → Rede → interface atual → Detalhes → Proxies; marque HTTP/HTTPS (ou SOCKS5), servidor = IP do host, porta = porta do app.
  • Outros dispositivos: preencha IP do host + porta nas configurações de proxy HTTP/HTTPS/SOCKS5 suportadas.

Назначение

Используйте это устройство в той же сети Wi‑Fi/точке доступа как шлюз LAN‑прокси; трафик других устройств проходит через этот хост (можно комбинировать с VPN хоста). Автообнаружение/шифрование отключены; клиент должен вручную вводить IP и порт хоста.

Шаги на хосте

  1. Откройте приложение, выберите режим прокси (рекомендуется «Автоопределение») и порт (по умолчанию 1080). Текущая версия направляет весь трафик через прокси.
  2. Убедитесь, что хост и клиенты в одной LAN и что порт не занят.
  3. Нажмите «Запустить шлюз» и держите foreground‑сервис активным.

Настройка клиента (ручные IP + порт)

  • Android: сведения о Wi‑Fi → Прокси = Вручную; хост = IP этого устройства, порт = порт приложения. Или заполните HTTP/HTTPS в приложениях; режим автоопределения обычно работает с HTTP/HTTPS/SOCKS5.
  • iOS: Настройки → Wi‑Fi → текущая сеть → Настроить прокси = Вручную; сервер = IP хоста, порт = порт приложения (HTTP/HTTPS).
  • Windows: Настройки → Сеть и Интернет → Прокси → Вручную; адрес = IP хоста, порт = порт приложения (HTTP/HTTPS); приложения с поддержкой SOCKS5 могут задать SOCKS5.
  • macOS: Настройки системы → Сеть → текущий интерфейс → Подробнее → Прокси; отметьте HTTP/HTTPS (или SOCKS5), сервер = IP хоста, порт = порт приложения.
  • Другие устройства: в полях прокси HTTP/HTTPS/SOCKS5 устройства/приложения укажите IP хоста + порт.

Kusudi

Tumia kifaa hiki kama lango la proksi la LAN katika Wi‑Fi/hotspot ile ile; trafiki ya vifaa vingine hupitia kwa host huyu (inaweza kuunganishwa na VPN ya host). Ugunduzi/usalama otomatiki umeondolewa; mteja lazima aingize IP na bandari ya host kwa mkono.

Hatua za host

  1. Fungua app, chagua hali ya proksi (inapendekezwa “Utambuzi otomatiki”) na bandari (chaguo-msingi 1080). Toleo la sasa hupitisha trafiki yote kupitia proksi.
  2. Hakikisheni host na wateja wako LAN moja na bandari haijatumika.
  3. Bofya “Anzisha Gateway” na uache huduma ya foreground iendelee.

Usanidi wa mteja (IP + bandari ya mkono)

  • Android: Maelezo ya Wi‑Fi → Proksi = Manual; host = IP ya kifaa hiki, bandari = bandari ya app. Au jaza HTTP/HTTPS ndani ya app; hali ya utambuzi otomatiki kawaida hufanya kazi na HTTP/HTTPS/SOCKS5.
  • iOS: Settings → Wi‑Fi → mtandao wa sasa → Configure Proxy = Manual; seva = IP ya host, bandari = bandari ya app (HTTP/HTTPS).
  • Windows: Settings → Network & Internet → Proxy → Manual; anwani = IP ya host, bandari = bandari ya app (HTTP/HTTPS); programu zenye SOCKS5 zinaweza kuweka SOCKS5.
  • macOS: System Settings → Network → kiolesura cha sasa → Details → Proxies; tiki HTTP/HTTPS (au SOCKS5), seva = IP ya host, bandari = bandari ya app.
  • Vifaa vingine: ingiza IP ya host + bandari kwenye mipangilio ya proksi ya HTTP/HTTPS/SOCKS5 inayotolewa.

วัตถุประสงค์

ใช้เครื่องนี้เป็นเกตเวย์พร็อกซี LAN ในเครือข่าย Wi‑Fi/ฮอตสปอตเดียวกัน เพื่อให้อุปกรณ์อื่นส่งทราฟฟิกผ่านโฮสต์นี้ (ใช้ร่วมกับ VPN ของโฮสต์ได้) ฟังก์ชันค้นหา/เข้ารหัสอัตโนมัติถูกนำออกไปแล้ว; ไคลเอนต์ต้องกรอก IP และพอร์ตของโฮสต์เอง

ขั้นตอนฝั่งโฮสต์

  1. เปิดแอป เลือกโหมดพร็อกซี (แนะนำ “ตรวจหาอัตโนมัติ”) และพอร์ต (ค่าเริ่มต้น 1080) เวอร์ชันปัจจุบันบังคับให้ทราฟฟิกทั้งหมดผ่านพร็อกซี
  2. ยืนยันว่าโฮสต์และไคลเอนต์อยู่ใน LAN เดียวกัน และพอร์ตไม่ถูกใช้งาน
  3. แตะ “เริ่มเกตเวย์” และให้บริการ foreground ทำงานต่อ

การตั้งค่าไคลเอนต์ (กรอก IP + พอร์ตเอง)

  • Android: รายละเอียด Wi‑Fi → Proxy = Manual; โฮสต์ = IP ของเครื่องนี้, พอร์ต = พอร์ตในแอป หรือกรอก HTTP/HTTPS ในแอป โหมดตรวจหาอัตโนมัติโดยมากใช้ได้กับ HTTP/HTTPS/SOCKS5
  • iOS: การตั้งค่า → Wi‑Fi → เครือข่ายปัจจุบัน → ตั้งค่าพร็อกซี = Manual; เซิร์ฟเวอร์ = IP โฮสต์, พอร์ต = พอร์ตในแอป (HTTP/HTTPS)
  • Windows: Settings → Network & Internet → Proxy → Manual; address = IP โฮสต์, port = พอร์ตแอป (HTTP/HTTPS); แอปที่รองรับ SOCKS5 สามารถตั้ง SOCKS5 ได้
  • macOS: System Settings → Network → อินเทอร์เฟซปัจจุบัน → Details → Proxies; ติ๊ก HTTP/HTTPS (หรือ SOCKS5), server = IP โฮสต์, port = พอร์ตแอป
  • อุปกรณ์อื่น: กรอก IP โฮสต์ + พอร์ตในช่องตั้งค่าพร็อกซี HTTP/HTTPS/SOCKS5 ที่อุปกรณ์/แอปมีให้

Layunin

Gamitin ang aparatong ito bilang LAN proxy gateway sa parehong Wi‑Fi/hotspot; dumaraan ang trapiko ng ibang device sa host na ito (puwedeng isabay sa VPN ng host). Inalis ang auto-discovery/encryption; kailangang ilagay ng kliyente nang manu-mano ang IP at port ng host.

Mga hakbang ng host

  1. Buksan ang app, piliin ang mode ng proxy (inirerekomenda ang “Auto-detect”) at ang port (default 1080). Sa kasalukuyang bersyon, lahat ng trapiko ay dumadaan sa proxy.
  2. Tiyaking nasa parehong LAN ang host at mga kliyente at hindi ginagamit ang port.
  3. Pindutin ang “Start Gateway” at panatilihing buhay ang foreground service.

Setup ng kliyente (manu-manong IP + port)

  • Android: Wi‑Fi details → Proxy = Manual; host = IP ng aparatong ito, port = port ng app. O punan ang HTTP/HTTPS sa mga app; karaniwang gumagana ang Auto-detect mode sa HTTP/HTTPS/SOCKS5.
  • iOS: Settings → Wi‑Fi → kasalukuyang network → Configure Proxy = Manual; server = IP ng host, port = port ng app (HTTP/HTTPS).
  • Windows: Settings → Network & Internet → Proxy → Manual; address = IP ng host, port = port ng app (HTTP/HTTPS); maaaring i-set ang SOCKS5 sa mga app na sumusuporta rito.
  • macOS: System Settings → Network → kasalukuyang interface → Details → Proxies; piliin ang HTTP/HTTPS (o SOCKS5), server = IP ng host, port = port ng app.
  • Ibang device: ilagay ang IP ng host + port sa mga setting ng HTTP/HTTPS/SOCKS5 proxy na sinusuportahan ng device/app.

Amaç

Aynı Wi‑Fi/erişim noktasında bu cihazı LAN proxy geçidi olarak kullanın; diğer cihazların trafiği bu host üzerinden gider (host VPN’iyle birlikte kullanılabilir). Otomatik keşif/şifreleme kaldırıldı; istemci host IP’si ve portu elle girmelidir.

Host adımları

  1. Uygulamayı açın, proxy modunu seçin (“Otomatik algıla” önerilir) ve portu belirleyin (varsayılan 1080). Mevcut sürüm tüm trafiği proxy üzerinden yönlendirir.
  2. Host ve istemcilerin aynı LAN’de olduğundan ve portun boş olduğundan emin olun.
  3. “Gateway’i başlat”a dokunun ve ön plan servisini açık tutun.

İstemci kurulumu (manuel IP + port)

  • Android: Wi‑Fi ayrıntıları → Proxy = Manuel; host = bu cihazın IP’si, port = uygulama portu. Ya da uygulamalarda HTTP/HTTPS alanlarını doldurun; Otomatik algılama modu genellikle HTTP/HTTPS/SOCKS5 ile çalışır.
  • iOS: Ayarlar → Wi‑Fi → mevcut ağ → Proxy’yi Yapılandır = Manuel; sunucu = host IP’si, port = uygulama portu (HTTP/HTTPS).
  • Windows: Ayarlar → Ağ ve İnternet → Proxy → Manuel; adres = host IP’si, port = uygulama portu (HTTP/HTTPS); SOCKS5 destekleyen uygulamalarda SOCKS5 ayarlanabilir.
  • macOS: Sistem Ayarları → Ağ → mevcut arayüz → Ayrıntılar → Proxyler; HTTP/HTTPS’yi (veya SOCKS5) işaretleyin, sunucu = host IP’si, port = uygulama portu.
  • Diğer cihazlar: Cihaz/uygulamanın desteklediği HTTP/HTTPS/SOCKS5 proxy alanlarına host IP + port yazın.

مقصد

اسی Wi‑Fi/ہاٹ اسپات پر اس ڈیوائس کو LAN پراکسی گیٹ وے کے طور پر استعمال کریں؛ دیگر ڈیوائسز کی ٹریفک اس ہوسٹ سے گزرے گی (ہوسٹ VPN کے ساتھ مل کر استعمال کیا جا سکتا ہے)۔ خودکار تلاش/انکرپشن ہٹا دی گئی ہے؛ کلائنٹ کو ہوسٹ کا IP اور پورٹ ہاتھ سے درج کرنا ہوگا۔

ہوسٹ کے مراحل

  1. ایپ کھولیں، پراکسی موڈ منتخب کریں (تجویز “آٹو ڈٹیکٹ”) اور پورٹ منتخب کریں (ڈیفالٹ 1080)۔ موجودہ ورژن ساری ٹریفک پراکسی سے گزارتا ہے۔
  2. یقین کریں کہ ہوسٹ اور کلائنٹس ایک ہی LAN پر ہیں اور پورٹ خالی ہے۔
  3. “گیٹ وے شروع کریں” پر ٹیپ کریں اور فورگراؤنڈ سروس کو جاری رکھیں۔

کلائنٹ سیٹ اپ (ہاتھ سے IP + پورٹ)

  • Android: Wi‑Fi تفصیل → پراکسی = Manual؛ ہوسٹ = اس ڈیوائس کا IP، پورٹ = ایپ کا پورٹ۔ یا ایپس میں HTTP/HTTPS درج کریں؛ آٹو ڈٹیکٹ موڈ عموماً HTTP/HTTPS/SOCKS5 کے ساتھ کام کرتا ہے۔
  • iOS: Settings → Wi‑Fi → موجودہ نیٹ ورک → Configure Proxy = Manual؛ سرور = ہوسٹ IP، پورٹ = ایپ پورٹ (HTTP/HTTPS).
  • Windows: Settings → Network & Internet → Proxy → Manual؛ address = ہوسٹ IP، port = ایپ پورٹ (HTTP/HTTPS)؛ SOCKS5 کو سپورٹ کرنے والی ایپس میں SOCKS5 سیٹ کریں۔
  • macOS: System Settings → Network → موجودہ انٹرفیس → Details → Proxies؛ HTTP/HTTPS (یا SOCKS5) منتخب کریں، server = ہوسٹ IP، port = ایپ پورٹ۔
  • دیگر ڈیوائسز: ڈیوائس/ایپ کی HTTP/HTTPS/SOCKS5 پراکسی سیٹنگ میں ہوسٹ IP + پورٹ لکھیں۔

Mục đích

Dùng thiết bị này làm gateway proxy LAN trong cùng Wi‑Fi/điểm phát; lưu lượng của thiết bị khác sẽ đi qua máy chủ này (có thể dùng kèm VPN của máy chủ). Tính năng tự động phát hiện/mã hóa đã bỏ; client phải tự nhập IP và cổng của máy chủ.

Các bước trên máy chủ

  1. Mở ứng dụng, chọn chế độ proxy (khuyến nghị “Tự động nhận diện”) và cổng (mặc định 1080). Phiên bản hiện tại đưa toàn bộ lưu lượng qua proxy.
  2. Đảm bảo máy chủ và client cùng LAN và cổng không bị chiếm.
  3. Nhấn “Khởi động gateway” và giữ dịch vụ nền trước hoạt động.

Cấu hình client (IP + cổng thủ công)

  • Android: Chi tiết Wi‑Fi → Proxy = Manual; host = IP của thiết bị này, port = cổng ứng dụng. Hoặc điền HTTP/HTTPS trong ứng dụng; chế độ tự động nhận diện thường hoạt động với HTTP/HTTPS/SOCKS5.
  • iOS: Cài đặt → Wi‑Fi → mạng hiện tại → Cấu hình proxy = Manual; máy chủ = IP host, cổng = cổng ứng dụng (HTTP/HTTPS).
  • Windows: Settings → Network & Internet → Proxy → Manual; địa chỉ = IP host, cổng = cổng ứng dụng (HTTP/HTTPS); ứng dụng hỗ trợ SOCKS5 có thể đặt SOCKS5.
  • macOS: Cài đặt hệ thống → Mạng → giao diện hiện tại → Chi tiết → Proxy; đánh dấu HTTP/HTTPS (hoặc SOCKS5), máy chủ = IP host, cổng = cổng ứng dụng.
  • Thiết bị khác: điền IP host + cổng trong phần cấu hình proxy HTTP/HTTPS/SOCKS5 mà thiết bị/ứng dụng hỗ trợ.
