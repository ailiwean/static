Network Share Data Security Statement

How Network Share protects data within the LAN proxy scenario.

Security principles

  • Local-only forwarding: traffic stays in the same LAN/hotspot; no cloud relay.
  • No payload logging: we do not inspect/store/upload proxied payloads or headers.
  • TLS end-to-end: HTTPS negotiated between client and destination; we do not terminate TLS.
  • Host VPN aware: if the host runs its own VPN, traffic may traverse it under host control.
  • Minimal permissions: network, foreground service/notification, vibration, storage; no contacts/photos/SMS/mic/precise location.
  • Third parties (optional): Firebase Crashlytics for crash metadata; Google AdMob for ads—no proxy content sent.

User controls

  • Clear app data or uninstall to remove local cache/config.
  • Disable crash reporting/ads (if available) to stop metadata/ad signals.
  • Use trusted Wi‑Fi, strong hotspot passwords, and keep OS/app updated.
  • Contact: wenyuanshuai@gmail.com

安全原则

  • 仅本地转发：流量只在同一局域网/热点内，不经过云中继。
  • 不记录内容：不检查、存储或上传代理报文内容/报文头。
  • TLS 端到端：HTTPS 由客户端与目标站点直接协商，我们不终止 TLS。
  • 主机 VPN 透明：如主机启用自有 VPN，流量可能在其控制下经过该 VPN。
  • 最小权限：仅网络、前台服务/通知、震动、存储，不访问通讯录/相册/SMS/麦克风/精确位置。
  • 第三方（可选）：Crashlytics（崩溃元数据）、AdMob（广告），不包含代理内容。

用户可控

  • 清除应用数据或卸载以删除本地缓存/配置。
  • 关闭崩溃上报/广告（如有入口）以停止元数据/广告信号。
  • 使用可信 Wi‑Fi、设置强热点密码，保持系统/应用更新。
  • 联系：wenyuanshuai@gmail.com
